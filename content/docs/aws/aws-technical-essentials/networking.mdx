## 네트워킹 소개

<Callout>
  네트워킹은 전 세계의 컴퓨터를 연결하여 서로 통신할 수 있도록 하는 방법입니다.
</Callout>

### 네트워킹 정의

- 전 세계의 컴퓨터를 연결하여 서로 통신할 수 있도록 하는 방법
- AWS에서는 글로벌 인프라가 있으며 데이터 센터, 가용 영역 및 리전을 사용하여 리소스 네트워크를 구축한다.

### 네트워킹 기본 사항

- 편지 보내는 일에 비유:
  - 정확한 주소를 바탕으로 편지를 전달하는 행위 = 네트워킹의 라우팅

### IP 주소

- 메시지를 라우팅하기 위한 주소
- 비트 0과 1의 조합을 사용한다.

### IPv4 표기법

- 일반적으로 IP 주소가 바이너리 형태로 표시되지 않고, 10진수 형식으로 변환되어 IPv4 주소로 표시된다.

| 항목            | 설명                                                                                 |
| --------------- | ------------------------------------------------------------------------------------ |
| **기본 형식**   | 32비트 주소 → **4개의 8비트 그룹**으로 나눔                                          |
| **표기 방식**   | 각 8비트 그룹을 **10진수로 변환**, 마침표(`.`)로 구분. 예를 들어, `192.168.1.30`     |
| **표기 예시**   | `11000000.10101000.00000001.00011110` → `192.168.1.30`                               |
| **CIDR 필요성** | IPv4 주소는 **네트워크 작업**을 위해 CIDR 표기와 함께 사용됨 (예: `192.168.1.30/24`) |
| **기억할 점**   | IPv4 주소는 **통신 대상 컴퓨터를 식별**하기 위한                                     |

### CIDR 표기법

- CIDR 표기법은 IP 주소 범위를 `시작 IP / 고정 비트 수` 형태로 나타내는 방식이다.
  - 예를 들어, `192.168.1.0/24` 에서 고정 비트 수는 `24` 이고 한 그룹당 8비트라 고정 비트 영역은 `192.168.1` 이며 나머지는 유동적이다.
  - 이 경우, 유동 비트가 8개이고 `2^8 = 256` 개의 IP 주소를 포함한다. 즉, `192.168.1.0` 부터 `192.168.1.255` 까지의 범위를 가진다.
- 슬래시 뒤 숫자가 커질수록 고정된 부분이 많아지고, 사용 가능한 IP 주소 수는 줄어든다.
- AWS에서는 CIDR 표기법으로 네트워크 크기를 지정한다.
  - 최소는 `/28` 로 16개의 IP를 제공하며, 최대는 `/16` 으로 65,536개의 IP를 제공한다.

## Amazon VPC

<Callout>
  중복성과 내결함성을 유지하기 위해 2개의 가용 영역에 구성된 서브넷을 2개 이상 생성합니다.
</Callout>

### Amazon VPC

- Virtual Private Cloud(VPC)는 데이터 센터의 기존 네트워크와 유사하게 AWS 클라우드에서 생성하는 격리된 네트워크다
